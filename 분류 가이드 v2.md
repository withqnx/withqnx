## 카테고리 (6개)

순서: `긍정 → 부정 → 중립/단순수령 → 양도/거래 → 교환/반품 → 배송관련불편`

| 카테고리 | 색상 | 정의 | 헷갈리는 것 |
|---|---|---|---|
| **긍정** | `#60a5fa` 파랑 | 상품 만족·칭찬·감동 | 단순 수령은 중립 |
| **부정** | `#ef4444` 빨강 | 상품 자체 불만 | 배송 문제는 배송관련불편 |
| **중립/단순수령** | `#22c55e` 초록 | 이유 없는 호평, 수령 알림, 기대 표명 | **단독만, 태그 빈배열** |
| **양도/거래** | `#16a34a` 진초록 | 구매자끼리 양도·판매·교환 | 회사 요구는 교환/반품 |
| **교환/반품** | `#fb923c` 주황 | 회사에 교환·환불 요청 | 구매자끼리는 양도/거래 |
| **배송관련불편** | `#c084fc` 보라 | 박스/지연/오배송/누락/포장 | 상품 자체 문제는 부정 |

### 중립/단순수령의 특별 규칙
- segments에 **혼자만** 올 수 있음
- hashtags는 **반드시 빈 배열 `[]`**
- 다른 카테고리가 함께 나올 만하면 중립을 제외하고 그쪽을 씀

### 양도/거래 · 교환/반품 + 부정 공존 규칙
**행동(양도·환불·반품)과 사유(불만)는 별개의 segment**로 분리.

| 후기 | segments |
|---|---|
| "양도합니다" 단독 | 양도/거래 |
| "사이즈 안 맞아서 양도합니다" | 양도/거래 + 부정 (#사이즈) |
| "환불해주세요" 단독 | 교환/반품 |
| "마감 별로라 환불해주세요" | 교환/반품 + 부정 (#마감) |
| "박스가 찌그러져서 교환해주세요" | 교환/반품 + 배송관련불편 |

### 절대 규칙
1. **`#배송` 금지** (배송관련불편 카테고리가 대체)
2. **중립의 hashtags는 빈 배열**
3. **"예쁘다" 단독은 #디자인 안 붙임**
4. **헷갈리면 안 붙이는 쪽**

---

## 신뢰도 (confidence)

| 범위 | 의미 |
|---|---|
| 0.9 ~ 1.0 | 단서 명확, 단일 해석 |
| 0.8 ~ 0.9 | 명확하지만 약간의 여지 |
| 0.6 ~ 0.8 | 애매함 → `needs_review: true` |
| < 0.6 | 매우 애매함 → `needs_review: true` 필수 |

**철칙: 헷갈리면 confidence ↓. 자신감 과장 금지.**

## 출력 JSON 스키마

```json
{
  "segments": [
    {
      "category": "긍정",
      "summary": "디자인 만족",
      "hashtags": ["#디자인"],
      "confidence": 0.9
    }
  ],
  "confidence": 0.9,
  "needs_review": false,
  "reasoning": "분류 이유 한 줄"
}
```

- 최상위 `confidence` = segments 중 최솟값
- `needs_review` = `confidence < 0.8` 이면 true
- `mixed` 필드는 더 이상 사용 안 함
- 후기 단위 `hashtags` 합집합은 후처리에서 자동 생성
